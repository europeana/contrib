Getting Started

The YUMA Annotation Server is built with Apache Ant. Use the default 'create-war' target to build the .war file.

The project comes with an included starter class that will launch the project in an embedded Jetty Web server (version 6.1.26). This way, you can work interactively on the code in your IDE without the need to constantly re-build and re-deploy. Please ensure that your IDE compiles the application classes to the /src/main/webapp/WEB-INF/classes directory. Then locate the class

at.ait.dme.yuma.server.bootstrap.StartAnnotationServer

in the /src/main/test folder, and launch it as a Java application. Per default, the Jetty will launch on port 8080. The YUMA Annotation Server will be available at http://localhost:8080/yuma-server. Modify the settings in the starter class if you want a different port or application root path. When developing, please note that changes in HTML templates and CSS will be effective immediately, while changes in Java classes require a restart of the Jetty before taking effect!

Dependencies

In case you want to hack on the source code of the YUMA Annotation Server, you may want to familiarize yourself with one or more of its library dependencies:

  • RESTEasy: JAX-RS implementation used for the API controllers (version 2.0.1.GA)
  • Hibernate: persistence framework for relational databases (version 3.6.0.Final)
  • Jena Semantic Web Framework: used for RDF (de-) serialization (version 2.6.4)
  • Rome: RSS feed generation (version 1.0)
  • Apache Wicket: Web framework used for the GUI (version 1.4.14)

Adding a New Input or Output Data Format

You can add new input and/or output formats to the YUMA annotation server by implementing a custom FormatHandler, along with your own subclass of the AbstractAnnotationController. FormatHandlers are responsible for parsing and serializing data formats. AnnotationControllers expose these formats through REST endpoints. You can find examples (for JSON, RDF, RSS, etc.) in the at.ait.dme.yuma.server.controller package. Be sure to read the JavaDoc in the code (especially in the base classes) for instructions and hints!

Adding Support for a New Storage Backend

You can add support for new types of storage backends by implementing your own custom subclass of AbstractAnnotationDB. Subclasses need to provide a set of methods for basic CRUD and query operations. You will find examples in the at.ait.dme.yuma.server.db package.

Please note that this is normally NOT required if you want to attach a new relational database! If your database is among the Hibernate-supported databases, you should only need to add the appropriate JDBC driver to the project, and edit the Hibernate configuration in the web.xml file.
